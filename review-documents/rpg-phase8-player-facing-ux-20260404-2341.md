## Goal

Create a comprehensive test suite for the player-facing UX layer covering:
- Player state management (ensure_player_state, set_current_scene, push_scene_history)
- Dialogue mode transitions (enter_dialogue_mode, exit_dialogue_mode)
- Journal management (update_journal_from_state) with dedup and cap at 200
- Codex management (update_codex_from_state) with capped buckets at 200
- Encounter view building (build_encounter_view) with bounded payloads
- 7 player API routes (state, journal, codex, objectives, dialogue/enter, dialogue/exit, encounter)

## Design Rules

1. Deterministic behavior - same input produces same output
2. Backward compatibility - existing player_state fields are preserved
3. Bounded data structures - journal capped at 200, scene history at 50
4. Integration with simulation state - player layer sits above simulation
