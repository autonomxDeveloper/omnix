This combines:

12.3 — Image cache / storage / replay-safe asset references
12.4 — Visual continuity system / appearance evolution tracking
12.5 — Image generation controls / moderation / fallback handling

It builds directly on your current Phase 12 backend, which already has visual_state, portrait request/result routes, and scene illustration request/result routes.

This phase keeps the same rules:

images remain presentation assets, not simulation truth
deterministic gameplay must not depend on asset generation output
asset references must be stable, bounded, and replay-safe
visual continuity changes must be explicit and inspectable
moderation/fallback must be explicit and provider-agnostic
High-level outcome

After this patch, your visual system will support:

stable asset registry entries instead of only loose URLs
cache keys / asset IDs for replay-safe references
appearance profile + evolution log for continuity
visual refresh reasons like injury, faction change, corruption, disguise, promotion
moderation / validation / fallback state for portrait and scene requests
request lifecycle with retryable failure states and blocked status

+def _safe_bool(value: Any, default: bool = False) -> bool:
+    if isinstance(value, bool):
+        return value
+    if isinstance(value, (int, float)):
+        return bool(value)
+    if isinstance(value, str):
+        text = value.strip().lower()
+        if text in {"1", "true", "yes", "y", "on"}:
+            return True
+        if text in {"0", "false", "no", "n", "off"}:
+            return False
+    return default

+def validate_visual_prompt(prompt: str) -> Dict[str, Any]:
+    prompt = _safe_str(prompt).strip()
