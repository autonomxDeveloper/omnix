Damage formula

Use deterministic formula like:

base_damage = weapon.damage
stat_bonus = strength_mod or dexterity_mod depending on weapon
skill_bonus = skill_level // 2
quality_bonus = weapon.quality_tier
crit_bonus = weapon.crit_bonus if crit else 0
final_damage = max(0, base_damage + stat_bonus + skill_bonus + quality_bonus + crit_bonus - target_armor)
Important

The LLM must not decide whether an attack lands.
It may decide:

weapon archetype suggestion at generation time
item flavor/stats when an item is created
optional move suggestions

But the authoritative action outcome must be deterministic engine logic.

Critical design rules for the coding LLM

Use these rules while implementing:

Simulation decides outcomes.
LLM may suggest item/archetype metadata, but never hit/miss/damage/xp totals.
Every narrated item interaction must match state.
No “you draw a sword” unless sword exists in inventory/equipment.
Every player action must map to stats and a skill family.
Even dialogue, stealth, hacking, investigation.
Skill progression is use-based.
Archery improves through bow use; persuasion improves through persuasion checks.
Generated item stats must be normalized and bounded.
Let AI flavor items, not break power scaling.
XP comes from structured sources only.
Enemy difficulty, quest rank, discovery rank, action difficulty.
Nearby NPCs are scene-scoped.
Known NPC registry is separate from active nearby cards.
Memory is compact in player UI, full in inspector.
