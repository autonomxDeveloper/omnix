Goal

Transform RPG flow from:

player acts
world replies

into:

world keeps moving
NPCs make decisions
NPCs talk to each other
NPCs talk to the player first
ambient incidents happen
the player can be interrupted
all of it remains deterministic, bounded, replay-safe
Design principles
1. Simulation truth first

Ambient activity must come from simulation state, not from the LLM inventing story.

Correct split:

simulation decides what happens
dialogue/narration decides how it is expressed
2. Server authoritative

The frontend should never create world events or NPC initiative.
It may only:

request idle advancement
subscribe to updates
render results
3. Bounded and replay-safe

Everything must be:

tick-based
recorded
capped
deterministic in ordering
fail-fast in replay when capture is missing
4. Player relevance filter

The world can be alive without spamming the player.
Only player-relevant activity should surface to the feed.

System overview

Build this in five layers:

Phase 1 — Ambient simulation advancement

Idle ticks advance the world even when the player does nothing.

Phase 2 — NPC initiative engine

NPCs decide to speak, confront, warn, recruit, gossip, or react without player prompting.

Phase 3 — Interruptions and delivery policy

Urgent events can interrupt; low-priority activity is queued or badged.

Phase 4 — Player-facing ambient feed

The UI renders autonomous activity distinctly from direct turn responses.

Phase 5 — Opening-aware ambient behavior

The new setup/opening data drives early idle activity, so the first 5–10 minutes feel authored, not generic.

Phase F — World Behavior Customization / Director Controls

This phase gives the player or creator control over how alive, aggressive, chatty, reactive, and story-directed the world feels.

Without this phase, even a good ambient system can feel wrong because different players want very different experiences:

some want constant interruptions
some want a quieter exploration game
some want heavy faction activity
some want NPC banter but not forced quest prompting
some want slow-burn realism
some want high-drama story pressure

So this phase should make the ambient system tunable, not one-size-fits-all.

Phase F — World Behavior Customization / Director Controls
Objective

Let the creator and/or player customize:

how often the world acts without input
how often NPCs initiate
how often interruptions happen
how strongly the opening/conflict is pushed
how much ambient dialogue vs world events appear
how aggressive quest prompting is
how reactive factions are
how much the simulation pressures the player
how noisy companions are
how “story directed” vs “sandbox autonomous” the experience feels

This should be available in a Settings / Director / World Behavior panel.

Design principle

These controls must bias the system, not override simulation truth.

Correct model:

settings affect salience, cooldowns, surfacing thresholds, pacing
settings do not directly force impossible events
settings do not let presentation invent truth

So:

“High NPC initiative” means lower cooldowns / higher candidate weights
not “always make NPCs talk every tick”
Where this belongs

You can expose it in two places:

Option A — Adventure Builder

Good for creator-facing setup defaults.

Put it under:

Setting panel
or a dedicated World Behavior / Director step
Option B — In-game Settings

Good for player live tuning during play.

This is valuable because a player may start wanting “high drama” and later switch to “quieter sandbox”.

Best approach

Support both:

builder default settings saved into the adventure
runtime session override settings for current playthrough
What should be customizable
1. Ambient world activity

How often non-player activity appears.

Field:

"ambient_activity": "low" | "medium" | "high"

Effect:

adjusts idle tick surfacing threshold
adjusts world-event emission frequency
adjusts coalescing sensitivity
2. NPC initiative frequency

How often NPCs speak or act first.

Field:

"npc_initiative": "low" | "medium" | "high"

Effect:

candidate salience bonus/penalty
cooldown duration scaling
per-scene chatter caps
3. Interruption intensity

How often the player gets interrupted.

Field:

"interruptions": "minimal" | "normal" | "frequent"

Effect:

interrupt threshold
badge vs interrupt cutoff
quiet-window strength
4. Quest prompting strength

How much the world proactively offers hooks/jobs/leads.

Field:

"quest_prompting": "off" | "light" | "guided" | "strong"

Effect:

enables/disables quest_prompt candidates
changes salience of opening/objective prompts
changes repeat cooldowns for lead-offers
5. Companion talkativeness

How much companions comment/react.

Field:

"companion_chatter": "quiet" | "normal" | "talkative"

Effect:

companion comment candidate weights
companion cooldowns
companion interruption permissions
6. Faction/world pressure

How strongly the world escalates while idle.

Field:

"world_pressure": "gentle" | "standard" | "harsh"

Effect:

faction pressure surfacing
escalation salience
urgency of hostile/world-event activity

This should bias surfacing first, not directly rewrite simulation logic unless you explicitly want that later.

7. Opening guidance strength

How strongly the system keeps pushing the opening conflict.

Field:

"opening_guidance": "light" | "normal" | "strong"

Effect:

opening bias in early initiative scoring
suppression of unrelated chatter
quest prompt salience tied to starter conflict
8. Story direction vs sandbox autonomy

The most important tuning axis.

Field:

"play_style_bias": "sandbox" | "balanced" | "story_directed"

Effect:

sandbox:
weaker opening/objective bias
more incidental world behavior
less proactive hook surfacing
story_directed:
stronger objective/opening bias
more NPCs steering toward meaningful conflict
more structured prompts and interruptions

This should be a bias, not a rail.

9. Content mix influence on ambient behavior

You already added desired_content_mix. Use it at runtime too.

Effect:

mystery-heavy → more rumors, witnesses, clues, quiet pressure
combat-heavy → more threat/warning/combat-start surfacing
intrigue-heavy → more faction messengers, offers, political whispers
exploration-heavy → more environmental/world-event surfacing

This is a strong bridge between builder setup and ambient runtime.
