Below is the full apply-this-patch spec for a Scene Continuity Engine that fits the architecture you now have:

deterministic
bounded
server-authoritative
scene-aware
compatible with your current initiative + scene-weaver + ambient feed setup

This adds multi-tick scene persistence and escalation so scenes no longer feel like one-off bursts.

Goal

Upgrade scenes from:

one tick
one burst
one exchange

to:

persistent across ticks
stateful
escalating
player-pull aware
able to resolve or spill into world events
Design summary
What a persistent scene should do

A scene should:

start from current scene-weaver output
survive across multiple idle ticks
remember topic, participants, tension, and progress
emit one or more new beats over time
escalate when ignored
optionally pull in player
terminate cleanly
What it should not do

It should not:

become simulation truth directly
create unbounded history
spam every tick
override impossible world state
require the LLM every tick
Core model

Add a bounded runtime structure:

"scene_runtime": {
  "active_scenes": [],
  "recent_scene_ids": [],
  "recent_scene_pairs": [],
  "last_scene_tick": -999,
}

Each active scene:

{
  "scene_id": "scene:bran:guard_1:opening_conflict",
  "scene_kind": "argument",
  "topic": "opening_conflict",
  "location_id": "gate_square",
  "participants": ["bran", "guard_1"],
  "primary_speaker_id": "bran",
  "secondary_speaker_id": "guard_1",
  "player_pull": "address_player",
  "opening_tied": True,
  "priority": 0.68,
  "tension": 0.52,
  "beat_cursor": 0,
  "max_beats": 4,
  "started_tick": 12,
  "last_advanced_tick": 12,
  "expiry_tick": 18,
  "status": "active"
}

Expected behavior after this patch

Before:

NPC scene appears once
disappears
does not evolve

After:

NPCs start a scene
scene continues next tick
tone escalates if ignored
player gets pulled in
final consequence may surface as a world event
finished scene cools down and is remembered

Example:

Tick 12: Bran and a guard argue quietly
Tick 13: tension rises, they sharpen their language
Tick 14: guard turns to player directly
Tick 15: argument spills into a visible disturbance / quest hook

That is the first truly ongoing world drama layer.

Acceptance checklist

You’re done when:

 scene_runtime.active_scenes persists between ticks
 one active scene can continue across 2–4 beats
 ignored scenes increase tension
 continuing scenes can pull in player
 resolved high-tension scenes can emit a world event
 active scene cleanup is bounded
 frontend scene blocks remain grouped across multiple ticks
