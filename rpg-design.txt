+        if "cruelty" in choice_tags:
+            loyalty_delta -= 0.04 if role in {"support", "ally"} else 0.02
+        if "victory" in choice_tags:
+            morale_delta += 0.03
+        if "retreat" in choice_tags:
+            morale_delta -= 0.03
+        if "greed" in choice_tags:

+        if "generosity" in choice_tags:
+            loyalty_delta += 0.02
+

+    return dict(v) if isinstance(v, dict) else {}
+

+def migrate_v6_to_v7(package: Dict[str, Any]) -> Dict[str, Any]:
+    package = dict(package or {})
+    state = _safe_dict(package.get("state"))
+    simulation_state = _safe_dict(state.get("simulation_state"))
+    player_state = _safe_dict(simulation_state.get("player_state"))
+    party_state = _safe_dict(player_state.get("party_state"))
+
+    companions = []
+    for comp in party_state.get("companions") or []:
+        if not isinstance(comp, dict):
+            continue
+        companions.append({
+            "npc_id": str(comp.get("npc_id") or ""),
+            "name": str(comp.get("name") or comp.get("npc_id") or "Companion"),
+            "hp": int(comp.get("hp") or 100),
+            "max_hp": int(comp.get("max_hp") or 100),
+            "loyalty": float(comp.get("loyalty", 0.5)),
+            "morale": float(comp.get("morale", 0.5)),
+            "role": str(comp.get("role") or "ally"),
+            "status": str(comp.get("status") or "active"),
+            "equipment": dict(comp.get("equipment") or {}),
+        })
+
+    party_state["companions"] = companions[:6]
+    party_state.setdefault("max_size", 3)
+    player_state["party_state"] = party_state
+    simulation_state["player_state"] = player_state
+    state["simulation_state"] = simulation_state
+    package["state"] = state
+    package["schema_version"] = 7
+    return package

Here’s a focused 9.3 likely bug pass / review checklist for Companion Narrative Integration, tuned to the architecture you’ve been building on top of 9.2’s companion AI/state system. The main new risk area is that narrative hooks often look “read-only” but quietly mutate state, drift out of determinism, or duplicate events.

Highest-risk bugs to check first
1. Companion interjections firing multiple times for the same scene

This is the biggest narrative-layer bug.

Look for:

scene start adds an interjection
dialogue entry adds another for the same scene
scene refresh/re-render adds a third
inspector/timeline rebuild accidentally re-triggers it

What to verify:

one interjection per intended trigger
repeated reads of the same payload do not append more history
there is a deterministic guard such as scene id + tick + companion id

Good pattern:

last = narrative_state.get("last_interjection") or {}
if last.get("scene_id") == scene_id and last.get("tick") == tick:
    return player_state
2. Narrative hooks mutating state during read-only view building

This is a classic failure in narrative systems.

Danger zones:

scene payload builders
dialogue payload builders
timeline summary builders
inspector render payload routes

What to verify:

build_companion_scene_reactions() can be pure if intended
record_companion_narrative_event() is only called from explicit mutation flows
“get scene” routes do not silently append history unless that is a deliberate state transition

Rule:

builders compute
recorders mutate

Never blur those.

3. Nondeterministic interjection selection

Narrative feels harmless, but replay consistency can break here fast.

What to verify:

companions are sorted before choosing among them
scene tags / reaction candidates are sorted
tie-breaks are explicit
no unordered dict traversal affects selected interjection text

Bad:

for comp in companions:

Good:

for comp in sorted(companions, key=lambda c: str(c.get("npc_id"))):
4. Downed or absent companions still narrating

You already guarded action systems in 9.2, but narrative often forgets those same guards.

What to verify:

status == "downed" companions do not interject
status == "absent" companions do not react to scenes
low-hp but active companions may react differently, but still deterministically
5. History growth unbounded

Narrative history will bloat saves if you do not cap it.

What to verify:

history is trimmed every time a new event is recorded
trimming is deterministic
oldest entries are removed first
nested event payloads are compact and serializable

Recommended invariant:

len(history) <= 20
Medium-risk integration bugs

What to verify:

migration uses package["state"]["simulation_state"], not a stale alternate path

migration manager imports migrate_v6_to_v7

engine version bump matches phase 9.3
7. Narrative state not normalized on load

Even if migration is correct, future dirty saves or hand-built tests can still create malformed state.

What to verify:

history is always a list
last_interjection is always a dict
last_scene_reactions is always a list

8. Scene reaction summaries diverging from companion state

You now have loyalty, morale, status, and role. Narrative stance must reflect current normalized companion state, not stale snapshots.

Another common problem:

full scene payload copied into history

What to verify:

history entries are compact
