## Phase 15.2 — Session/Package Round-Trip Hardening

### Purpose
Make session export/import safe and canonical with validation, normalization, deterministic ordering, and round-trip tests.

### Files Changed
- **`src/app/rpg/session/package_bridge.py`** — Complete rewrite with:
  - `validate_package_payload()` — validates schema version, session manifest id, structure
  - `_normalize_manifest()` — deterministic manifest normalization
  - `_normalize_visual_state_for_export()` — sorted visual assets/requests, caps (100 requests, 200 assets), excludes queue/job runtime state
  - `_normalize_memory_state_for_export()` — sorted actor memory by strength, capped at 50 entries per actor, rumor sorting
  - `session_to_package()` — deterministic export with sorted installed_packs
  - `package_to_session()` — validates before import, returns `{ok, session/errors}`, backward-compat kwargs
- **`src/app/rpg/api/rpg_presentation_routes.py`** — Updated export/import routes to use validation result

### Tests (28 total)
- **Unit (15):** `test_phase151_session_package_bridge.py` — round-trip, validation, normalization, caps, edge cases
- **Functional (5):** `test_phase152_functional.py` — full round-trip, idempotency, validation blocks
- **Regression (8):** `test_phase152_regression.py` — None inputs, non-dict fields, schema v0, queue exclusion, caps

### Acceptance
- ✅ Package import rejects malformed payloads
- ✅ Round-trip tests pass
- ✅ Queue/job runtime state excluded from exports
- ✅ All payloads bounded and deterministically ordered
